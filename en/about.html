Technologies used:

Unity

I made multiple projects in this engine, including my NPC game (later Blacksmiths Forge) which I stopped developing but I would want to get back at it. I was doing lots of simple projects, some of which I have unfortunately lost. There are projects which I work on right now using unity.

Godot
Postgre DBMS

Used this database for the fullscale website with authentication, Front-End, Back-End, nginx and other. Very popular opensource database system, cross-platform. It allows for complicated queries which I had used in the mentioned before website.

Modern web framework which I have used for the Front-End of a fullscale website with different features. It allowed me to intergrate the database and backend easily with their features, plugins and other.

SvelteKit
Node

Node Runtime which I used for the backend of a website with expressjs and other libraries like postgredb connectors, pools etc., in the mentioned fullscale website it was running privately and was used by the frontend. There are better options for a runtime hosting backend, like actix-web and other frameworks written in generally faster languages.

Programming Languages:

  • C#

    I have done plenty of projects in C#, that includes working with C# on Unity games, and on Godot games. I made libraries like BinPack (serializer) using C# and many other smaller console apps. I often use it to tinker with things.

  • C++

    One of the most recent languages I have learned, I used it to develop HarmoniaUI - a New UI System for Godot (a bit like websites). I would love to work with it more, I very much like it.

  • Javascript/Typescript

    Mostly used this language for scripts on websites and backend servers, it's a very universal language and can be used to develop countless things. I used TypeScript in a quite big website project which included a Database, Front-End server, Back-End server, nginx web server for managing all servers.

  • Python

    Tinkering and other, I don't use this language for big/complex projects but I used it for short projects, and maybe mid-sized projects like discord bot. It's a nice language to quickly develop something

  • Rust

    I used this language for the MasterVault password manager and a backend server. I would say it's a nice language and has a pretty unique syntax, but developing times are unforgiving, that might be an advantage as its said to be very safe

  • Java

    Language I used to develop HydrateMe app on android using the android studio. It's similiar to C# with some key differences in syntax and specific features (like enums). It's popular in backend apps/microservices sphere with its frameworks like Spring Boot
